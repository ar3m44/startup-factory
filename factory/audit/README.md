# Audit Directory

Эта директория содержит аудит лог всех событий в фабрике.

## Структура

Каждая запись — это файл формата `AUDIT-YYYY-MM-DD-HH-MM.md`, где:
- `YYYY-MM-DD` — дата события
- `HH-MM` — время события (часы-минуты)

## Пример

```
factory/audit/
  AUDIT-2025-01-15-14-30.md  — Запуск продукта V-2025-001-quickpoll
  AUDIT-2025-01-16-09-15.md  — Первая транзакция в V-2025-001-quickpoll
  AUDIT-2025-01-20-18-45.md  — Закрытие продукта V-2025-001-quickpoll
  ...
```

## Типы событий

- `venture_created` — создан новый продукт
- `venture_launched` — продукт запущен в production
- `venture_killed` — продукт закрыт
- `task_completed` — задача выполнена
- `payment_received` — получена оплата
- `milestone_reached` — достигнут milestone
- `decision_made` — принято важное решение
- `risk_identified` — выявлен риск

## Создание записи

Используй шаблон из `factory/templates/AUDIT_ENTRY_TEMPLATE.md`.

## Зачем это нужно

Аудит лог позволяет:
- Отслеживать все важные события
- Анализировать успехи и ошибки
- Принимать решения на основе данных
- Вести историю развития фабрики
