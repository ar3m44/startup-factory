# TASK-0010: Sanitize Hidden/Bidi Unicode + Add CI Check

**Priority**: P0
**Type**: Security/Maintenance
**Dependencies**: none

## Context

Some markdown files may contain hidden bidirectional Unicode characters (U+200B, U+FEFF, U+202A-U+202E, etc.) that can cause security issues and confusing diffs. GitHub shows warnings for these. We need to:
1. Scan and remove all hidden/bidi characters from existing files
2. Add a CI check to prevent them from being reintroduced

## Scope

- All `.md` files in `factory/`
- All `.md` files in `ventures/`
- All `.ts`, `.tsx`, `.js`, `.mjs` files in `src/` and `scripts/`
- CI workflow update

## Non-Goals

- Do not modify node_modules
- Do not change file content meaning (only remove invisible chars)
- Do not add complex tooling (simple grep/sed is fine)

## Description

1. Create a script `scripts/check-unicode.mjs` that:
   - Scans specified directories for hidden Unicode
   - Reports files with issues
   - Has a `--fix` flag to auto-remove them
   - Exit code 1 if issues found (for CI)

2. Run the fix on existing files

3. Add to CI workflow (`.github/workflows/ci.yml`):
   ```yaml
   - name: Check for hidden Unicode
     run: node scripts/check-unicode.mjs
   ```

## Acceptance Criteria

- [ ] Script `scripts/check-unicode.mjs` exists and works
- [ ] All existing files are clean (no hidden Unicode)
- [ ] CI includes Unicode check step
- [ ] CI fails if hidden Unicode is introduced

## Test Plan

1. Run `node scripts/check-unicode.mjs` - should pass
2. Manually add a zero-width space to a file
3. Run script again - should fail
4. Run with `--fix` - should clean the file
5. Run `npm run build` - should pass

## Rollback Plan

- Revert the CI workflow change
- Delete the script
